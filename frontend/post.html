HOW TO: update all npm packeges in your project at once

Jan 30, 2016 на 15 мин 3

Is there a quick and easy out-of-the-box way to update all of the npm packages in your project? Yes. Does it quite work as you expect? Not always. For those in camp TL;DR, if you really want to take a risk and update all your packages to their latest version at once with relative ease, write a quick node script(source below) to rewrite all the versions listed in package.json and then run it as part of an npm script.

How can I check to see which packages are out of date?

npm outdated

The outdated command will list any packages that are lower than the latest available release.

Is there not an npm command to upgrade these?

npm update --save/--save-dev

Yes. That is it. It will only update packages so far though. The “update” function respects semver. For example, if I have a package which is at version 1.3.5, but the latest version is 3.0.5, the package would only update to the highest version of 1 major, 1.9.9 for instance if that was the highest version prior to 2.0.0. There is reason for this. It makes sense. The idea being that you want to avoid updating to packages that may break your project/cause conflicts. Restricting the power of npm update promotes having to manually update all of your dependencies in the package.json file. Doing so one by one methodically will make it easier to spot the culprit if things stop working.
